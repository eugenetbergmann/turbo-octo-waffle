## Hard rules — reject any code that violates these
MO picklist table: WO010302 (NOT WO010033)
MO demand qty: QTYPENDING (NOT QTYREMAININGTOPOST, REQD_QTY)
MO active filter: MANUFACTUREORDERST_I IN (2, 3) (NOT NOT IN (5, 6))
MO date: STRTDATE (NOT STARTDATE)
MO qty: ENDQTY_I (NOT QTYENTERED)
PO header status: POSTATUS (NOT PURCHSTAT)
PO line status: POLNESTA (NOT missing)
PO open qty: QTYUNCMTBASE (NOT QTYORDER - QTYRCVD - QTYCANCE)
Lot master qty: QUANTITY (NOT QTY)
Item-vendor xref: IV00103 (NOT POP60000)
Vendor master: PM00200 (assumed exists)
Inventory qty: QTYONHND
Item class filter: ITMCLSCD NOT LIKE '20%'
Inactive filter: INACTIVE = 0
NONINVEN filter: NONINVEN = 0

## Status codes
MO: 0=void, 1=open, 2=released(YES), 3=in-process(YES), 5=closed, 6=cancelled, 7=complete, 8=historical
PO header/line: 1=new, 2=released, 3=change-order, 4=received, 5=closed(NO), 6=cancelled(NO)

## Architecture principles
MED/GP = source of truth. Never write back.
Carlsbad = analytical layer. SQL Agent only.
State tables refreshed on schedule. No real-time triggers.
Snapshots append-only. Never update or delete.
Every proc ends with usp_rep_log.
No SELECT *. Explicit columns only.
No magic numbers. Named config CTE.
TRY_CAST over ISNUMERIC.
NONINVEN = 0 on all receipt/cost queries.
ALTER VIEW only via SSMS. View Designer banned.
No Friday deploys.
